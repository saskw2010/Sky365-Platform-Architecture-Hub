ADR-0002: Canonical Controller Identity and Intermediate Model

Proposed - 2026-07-17

Context

SKY365 currently consumes Code On Time XML, file-based dynamic JSON schemas/catalogs and database-backed System.SysControllers. The runtime successfully bridges these models, but identity fields and preservation guarantees are not yet uniform.

Observed evidence

  • Catalog deduplication is effectively based on moduleKey + schemaName.
  • Runtime resolution also depends on system, module, schema and optional profile context.
  • SystemGroup values in the unified catalog include both technical keys and display labels.
  • Code On Time imports map fields, views, lookups and actions through more than one conversion path.
  • The file-based schema population is much larger than the unified discovery catalog.

Proposed decision

Adopt a canonical Controller identity tuple of systemKey/moduleKey/schemaKey. Treat titles and SystemGroup display labels as localized presentation metadata, never as identifiers. Define a versioned Controller Intermediate Representation (CIR) that both Code On Time XML and authored JSON compile into before runtime consumption.

Required CIR properties

  • Stable identity, source revision and provenance.
  • Entity/table/query identity and connection policy reference.
  • Fields, keys, types, formats, validation and visibility.
  • Lookups, master-detail and many-to-many relationships.
  • Views, sections, categories, actions and profiles.
  • Roles, permissions and policy hooks.
  • Workflow, API, report, dashboard, tool and semantic links.
  • Unsupported-source constructs and conversion warnings, so lossy import is visible.

Consequences

  • Catalog grouping, URLs, permissions and graph identities become deterministic.
  • Legacy XML remains a supported source without becoming the runtime source of truth.
  • The two existing Controller persistence paths can converge on behavior without an immediate destructive migration.
  • A compiler and conformance suite become required platform components.

Alternatives

Keep title-based SystemGroup identity: rejected because localization and naming changes destabilize routing. Make XML the canonical runtime format: rejected because modern JSON metadata extends beyond the legacy format. Replace all existing models immediately: rejected because it would create unnecessary migration risk.

Status and roadmap

This ADR remains Proposed until a field-loss matrix, identity migration impact report and golden Controller conformance tests are completed. Related: Code On Time architecture and Controller and Screen Registry.
