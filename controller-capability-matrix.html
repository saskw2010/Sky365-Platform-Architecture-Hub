Interpretation

The Controller is the central application metadata model, but it is not a monolith. Direct capabilities consume the screen schema itself. Linked capabilities maintain their own metadata and reference Controller identity. Consumers adapt Controller metadata for another runtime. Planned capabilities currently use the Controller only as a potential context source.

Architecture rule

Controller CIR = stable application contract |-- direct compilers: UI / CRUD / API description / tools |-- linked registries: workflow / dashboard / report / integration `-- derived graphs: semantic / impact / AI context All branches inherit canonical identity + permissions + provenance.

Roadmap

For each capability define whether it is Direct, Linked, Consumer or Planned; prohibit undocumented copying of Controller fields into another metadata store; verify permission and source revision propagation.
