HTML Documentation Conformance Audit

Semantic audit of purpose, architecture/visualization, evidence examples, relationships, roadmap and cross references by page type.

Conformance model

The mission requirements apply to architecture content pages. A table is a valid visualization when it explains exact mappings better than a flow diagram. Evidence rows, source-derived counts and traced request paths are valid examples. Navigation indexes, chronological ledgers and ADRs use specialized templates rather than artificial tutorials.

Discovery evidence → Reference / Explanation pages → visual relationship + concrete evidence → gaps/roadmap → cross references; major decision → ADR lifecycle

Validation and exceptions

  • All artifacts are HTML.
  • Every content page has a stated purpose in title/header or opening section.
  • Every content page contains a relationship diagram, mapping table, lifecycle or equivalent smallest useful visual.
  • Concrete source paths, counts, properties or runtime paths serve as examples.
  • Every content page includes future gaps/roadmap and cross references.
  • Specialized governance pages follow their own explicit templates; adding fake examples would reduce clarity.

Future roadmap

Re-run this audit when pages are added. A new architecture content page must identify its type, evidence scope, visualization, concrete example, roadmap and inbound/outbound links before the Hub can call it conformant.
