Legacy sky365ERP architecture

The project configuration identifies a Code On Time Web Site Factory application with generated Controllers and pages, SQL Server data access, membership, reports and Touch UI. Executable framework classes include:

  • IDataController, DataControllerBase, Controller factory/configuration and filtering/export logic;
  • business rules and action handlers;
  • application services, membership/security and OAuth infrastructure;
  • REST service/client infrastructure;
  • report handlers and report XML;
  • workflow resources/register types;
  • Controller data-source controls and generated UI assets.

These are legacy Code On Time runtime capabilities. They must not be counted as modern SKY365 engines unless a current runtime counterpart is independently verified.

Interpretation

  • The current metadata population is not a simple one-to-one migration of the inspected legacy ERP Controller set.
  • The large legacy XML count includes generated/copy populations and must not be reported as 6,380 distinct business screens.
  • The 13 shared names prove limited continuity, but field/view/action equivalence still requires structural comparison.
  • The current platform likely aggregates multiple Code On Time/database lineages.
  • The empty API repository means current API architecture must be documented from Sky365Offers, not inferred from the repository name.

Security evidence

The inspected legacy project configuration contains an embedded database connection secret. This Hub intentionally does not reproduce it. The repository should be treated as sensitive historical material until credentials are rotated and history exposure is assessed.

Migration and future roadmap

  1. Build a normalized legacy Controller inventory from the primary app/controllers set.
  2. Classify generated/copy folders separately.
  3. Match Controllers structurally by table/query, fields and relationships—not name alone.
  4. Select golden legacy Controllers for fidelity tests.
  5. Record Adopt / Adapt / Archive decisions per Controller.
  6. Never import credentials or machine-specific configuration into the Architecture Hub.
