4. Lookup definition

A lookup can use a legacy single source or an explicit Primary/Fallback contract. Metadata includes source/schema/controller/table identity, value/text/secondary fields, display template, quick-add schema, lookup type and a copy map.

The runtime validates that explicit contracts contain endpoint identity for both Primary and Fallback. Legacy shapes are accepted with a warning. This is a live migration boundary rather than one uniform contract.

5. Views, sections and actions

  • Sections: title, description, order and reusable layout metadata.
  • Views: name, labels, type, source/custom SQL, description, fields and actions.
  • Actions: name/ID, label/icon/color, type/endpoint/execute target, confirmation, roles, condition and payload mode.

These definitions drive presentation and executable interaction, but not every Code On Time import path preserves them.

6. Schema profiles

DynamicSchemaProfile overlays one base schema without copying it. It can rename the screen, select print template, add filters/defaults/details, choose list/form fields and override labels, placement, visibility, required/read-only state and maximum length.

This makes one Controller reusable across personas and workflows, but profile identity must remain part of runtime context and cache keys.
