Purpose: reconcile the Knowledge Hub target vocabulary with current implementation evidence.

Interpretation: “implemented subsystem” means meaningful executable source exists. It does not mean production-ready. “Partial” means useful primitives exist but the complete, governed engine boundary is not yet proven.

Platform engines outside the C01-C26 vocabulary

Verified implementation: Metadata/Controller Catalog, Dynamic UI, CRUD Runtime, Persistence, Workflow, Dashboard/BI, Report, Integration, Business Rule/Validation and Metadata Graph.

Partial boundaries: API generation, Search, Security/Permission, Notification, File, AI/Tools and Semantic Layer.

Not yet established as complete engines: Worker, Scheduler, Plugin, Extension, Vector Sync, OCR, Voice and Image Intelligence.

Relationships and roadmap

Engine maturity must be revised only after code/runtime evidence changes. Next audits will split each verified or advanced-partial entry into purpose, responsibilities, inputs, outputs, metadata, dependencies and extension points. See Code On Time Controller, Engine Catalog and Knowledge synthesis.
