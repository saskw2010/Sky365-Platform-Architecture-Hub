Planned and In-Progress Engine Roadmap

Purpose: define what must become true before a planned capability is promoted to an implemented engine.

Maturity gates

Documented only -> Prototype (demonstrable path) -> Partial implementation (executable components) -> Implemented engine (owned boundary + metadata + runtime) -> Operationally verified (tests + telemetry + failure evidence) -> Production-ready (security + scale + deployment + support evidence)

Recommended sequence

  1. Canonical Controller identity and CIR/compiler.
  2. Catalog coverage and golden screen runtime verification.
  3. Cross-engine Security/Permission contract.
  4. API contract compiler and safe Action Core expansion.
  5. Semantic Dictionary on Metadata Graph.
  6. Prompt/Context compiler and Hybrid RAG evaluation.
  7. Vector Sync only after stable identity, provenance and ACLs.
  8. Worker/Scheduler and plugin lifecycle after observability standards are shared.

Decision rule

No item should be called an Engine merely because a class, screen, blueprint or landing page uses the name. Promotion requires an owned runtime boundary, explicit metadata contract, inputs/outputs, dependencies, extension points, failure semantics and verification evidence.
