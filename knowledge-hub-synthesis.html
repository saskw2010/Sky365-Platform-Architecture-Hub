Sky365 Knowledge Hub Synthesis

Purpose: preserve strategic knowledge while separating target architecture from implemented evidence.

Source role

Sky365-knowledge-hub is the canonical strategy and knowledge operating system. It defines platform pillars, 26 core capabilities, evidence statuses, roadmaps and investment programs. It is authoritative for intent and vocabulary, but explicitly states that implementation claims must be checked against code or runtime evidence.

Architecture relationship

Knowledge Hub |-- vision / capability vocabulary / roadmaps |-- ADRs / blueprints / research / visual prototypes `-- source manifests and evidence policy | candidate claims v Architecture Discovery |-- repository + commit |-- code / metadata / database / route evidence |-- runtime or test evidence `-- maturity decision | validated finding v Platform Architecture Hub `-- engine catalog + Controller model + ADR + gaps

Status vocabulary adopted

StatusRequired evidence
Verified implementedExecutable code plus an identifiable runtime path; production-ready requires stronger runtime/deployment evidence.
Partial implementationSome executable components exist but the end-to-end boundary or coverage is incomplete.
PrototypeA demonstrable experiment or experience without production lifecycle evidence.
Documented onlyBlueprint, ADR, research or roadmap without executable platform evidence.
UnverifiedMentioned but not yet checked against the authoritative source scope.

Major programs discovered

  • Repository and Runtime Discovery.
  • Schema-Driven Platform Foundation: canonical schema, semantic dictionary, module/screen/workflow/report registries and legacy bridges.
  • Safe Agentic Execution: Action Core, policy, approvals, permissions, traces, verifiers and rollback.
  • Multimodal Intelligence: documents/OCR, voice, video, avatars and media.
  • Trust, Security and Monitoring.
  • Developer, Skill, Prompt, MCP and Evaluation platform.

The current engine maturity overlay is maintained in the Engine Maturity Matrix. The central legacy bridge is documented in Code On Time Controller Architecture.

Roadmap

Continue mapping every capability ID to exact source paths and runtime gates; create ADRs when a capability is promoted or reclassified; keep prototype, product and engine terminology distinct.
