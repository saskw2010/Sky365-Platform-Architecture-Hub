مسار النقاش — بالترتيب

الترتيبموضوع النقاشالقرار المطلوبالحالة
1 — التالي الآنQ002: Converter وSQL/Query Commandsهل نحفظ SQL خامًا، أم نحوله إلى Query/View Metadata مهيكلة وآمنة، وكيف نسجل conversion loss؟جاهز للنقاش
2Q003: Semantic Mapperهل يعيش كجزء من Controller Compiler، أم Metadata Graph/Semantic Engine، وما علاقة mapping بالحقول والـlookups؟مسجل
3Q004 + Q007: Worker/Scheduler وOffline/Syncتعريف Worker JSON وRunner/Queue/Retry/Schedule، ثم local store/outbox/sync/conflict contract المطلوب للـPOS والعمل دون اتصال.Offline foundation اكتُشف؛ الحدود الكاملة مفتوحة
4Q005: Shared Data Sources + Dashboard / BI / Reportsاعتماد DataSourceDefinition مشتركة يصممها EasyQuery وتستهلكها Grid/Report/Dashboard؛ ثم تحديد مكان Dashboard JSON وtags والـwidgets والـbindings.دور EasyQuery الموسع مثبت من المالك؛ العقد التفصيلي مفتوح
5UI Engine contractGroup/Component Registry/Layout/Theme tokens/Renderer profiles وصلاحيات العرض؛ دون تنفيذ المصمم الآن.الحدود الأولية معتمدة
6العشرة Core Enginesمراجعة كل محرك: purpose، metadata، runtime، services، lifecycle، dependencies وextension points.يلزم اعتماد المالك
7التسعة Partial Enginesتحديد الموجود فعليًا وما يلزم لترقيته: Security، API، AI/Tools، Search، Worker، Audit، Notification، File/Media، Video/Avatar.لم يبدأ سؤالًا بسؤال
8الستة Missing Enginesتحديد: مطلوب الآن، Roadmap لاحق، أم غير مطلوب: Scheduler، Plugin، Extension lifecycle، Vector Sync، OCR، Voice/Image.لم يبدأ
9Cross-engine architectureالهوية المشتركة، Security inheritance، Event/Audit contract، metadata versioning، والفصل بين Engine وService/Builder.بعد مراجعة القائمة
10إغلاق Goal 1إصدار ADRs النهائية، Target Architecture، dependency map، والأولويات المعتمدة.غير مكتمل
11 — لاحقًا فقطGoal 2: Implementationتحويل القرارات المعتمدة إلى epics وخطة تنفيذ واختبارات، بعد موافقة صريحة جديدة.غير مصرح به الآن

طريقة العمل في كل بند

  1. Codex يعرض الوضع الحالي المثبت من المصدر.
  2. المالك يشرح القصة: الحالي، الطبيعي، والهدف.
  3. نسجل الانحراف والأسئلة دون افتراض.
  4. نقترح بدائل قليلة مع آثارها.
  5. المالك يعتمد القرار أو يؤجله.
  6. نحدّث الـHub وADR عند القرار الكبير، ثم ننتقل إلى بند واحد فقط.
